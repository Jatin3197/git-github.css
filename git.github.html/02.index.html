git workflow between this three -working directory,staging area and repository

Git workflow is a set of steps that developers follow to manage and track files efficiently using git,is a famous version control system.It includes this three components like the working directory,staging area and repository.

Working Directory--

The working directory is a local file system to create a file ,edit a file and maintain your project files. When you initiate a Git repository in a directory, it becomes a part of the working directory.

Staging area--

Staging area is present between a working directory and repository.If you run the git Add command in the terminal then you are present to the staging area and ready to commit in the files.Before to commit the files is a staging area. Between add and commit command is present to a staging area.

Repository--

The repository is also known as git repository.Git permanently stores commited snapshots of your project or files. In the git Repository you can rename,edit,branches,tags and complete history of changes.
